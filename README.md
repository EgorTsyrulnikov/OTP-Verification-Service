# Promo IT OTP Verification Service

Простое и удобное backend-приложение для защиты операций с помощью одноразовых кодов (OTP). Разработано для автоматической генерации, рассылки (через Email, SMS, Telegram, файл) и валидации кодов.

## Особенности проекта
* **Стек технологий:** Java 17, Spring Boot (MVC), PostgreSQL 17, Maven.
* **База данных:** Взаимодействие с базой данных реализовано на **чистом JDBC** без использования ORM (Hibernate/JPA) в соответствии с ТЗ. Таблицы и начальная конфигурация создаются автоматически при старте приложения через класс `DatabaseInitializer`.
* **Архитектура:** Строгое разделение на слои API (Controllers), Сервисов (Business Logic) и DAO (JDBC-операции).
* **Безопасность:** Регистрация, хеширование паролей (SHA-256) и аутентификация через short-lived токены с проверкой ролей (ADMIN, USER) через `AuthInterceptor`. Поддерживается максимум один Администратор.
* **Фоновые задачи:** Автоматическое проставление статуса `EXPIRED` для просроченных кодов каждую минуту (`@Scheduled`).

## 🛠️ Как запустить и протестировать сервис

### 1. Подготовка базы данных
Убедитесь, что у вас установлен **PostgreSQL 17** и запущен на стандартном порту `5432`.
В `src/main/resources/application.properties` прописаны доступы по умолчанию. Для проверки приложение использует стандартную базу `postgres` (пользователь `postgres`, пароль `postgres`), чтобы вам не пришлось создавать отдельную базу вручную. Если вы используете другие учетные данные, измените их в этом файле.
*Таблицы (`users`, `otp_config`, `otp_codes`) создадутся сами при первом запуске!*

### 2. Настройка каналов связи
В папке `src/main/resources/` лежат файлы конфигураций для каналов отправки:
* `email.properties` - для Email (Angus Mail). Пропишите свои SMTP-данные.
* `sms.properties` - для SMS (JSMPP). Для тестирования запустите локальный эмулятор `SMPPsim`.
* `telegram.properties` - для Telegram (Java HttpClient). *В файле уже указан токен тестового бота. Для проверки генерации можете указывать любой `chat_id` (например, "123456") в качестве получателя — приложение корректно отправит запрос к Telegram API, залогирует ошибку (если чат не найден), но успешно продолжит работу.*

### 3. Запуск приложения
Запустите приложение из корневой папки с помощью Maven:
```bash
mvn spring-boot:run
```
*(Приложение запустится на порту `8080`)*

## 🚀 Сценарии использования и API-эндпоинты

### 1. Регистрация и Аутентификация
* **POST `/api/auth/register`** — Регистрация.
  ```json
  { "login": "admin", "password": "password", "role": "ADMIN" }
  ```
  *(Примечание: второго Администратора зарегистрировать нельзя).*
* **POST `/api/auth/login`** — Авторизация.
  ```json
  { "login": "admin", "password": "password" }
  ```
  *В ответ вы получите токен, который нужно передавать в заголовке `Authorization: Bearer <ваш_токен>` во всех следующих запросах.*

### 2. Панель Администратора (Требуется токен ADMIN)
* **PUT `/api/admin/config`** — Настройка времени жизни и длины OTP.
  ```json
  { "lifetimeSeconds": 60, "length": 4 }
  ```
* **GET `/api/admin/users`** — Получение списка всех пользователей (кроме админов).
* **DELETE `/api/admin/users/{id}`** — Удаление пользователя (и каскадно всех его OTP-кодов).

### 3. Панель Пользователя (Требуется любой токен)
* **POST `/api/otp/generate`** — Генерация OTP.
  ```json
  { 
    "operationId": "op_test_123", 
    "channel": "FILE", 
    "destination": "user_identifier" 
  }
  ```
  *Доступные каналы (`channel`):* `FILE`, `TELEGRAM`, `EMAIL`, `SMS`. 
  Если выбрать `FILE`, код будет сохранен в файл `otp_codes.txt` в корне проекта (удобно для быстрой проверки). При выборе `TELEGRAM` параметр `destination` должен содержать `chat_id` пользователя.

* **POST `/api/otp/validate`** — Валидация кода.
  ```json
  { "operationId": "op_test_123", "code": "1234" }
  ```
  *Если код верный, активный и его срок действия не истек, вернется `{"isValid": true}`.*
